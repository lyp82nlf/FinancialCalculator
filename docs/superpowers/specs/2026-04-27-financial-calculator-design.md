# 个人财务分配面板 — 设计规范

**日期：** 2026-04-27  
**状态：** 已审批

---

## 概述

一个纯前端单页应用，帮助用户将每月可支配收入按照自定义分类和比例进行分配，并通过里程碑卡片展示长期复利资产预测。无后端、无接口调用，所有数据持久化在浏览器 `localStorage` 中。

---

## 技术选型

| 项目 | 决策 |
|------|------|
| 实现方式 | 单个 `index.html` 文件，内嵌 CSS + JS |
| 数据持久化 | `localStorage`（key: `finance-config`） |
| 图表/可视化 | 纯 CSS（进度条、里程碑卡片），无第三方库 |
| 依赖 | 零依赖，无需构建工具，双击即可运行 |

## 页面结构（单页滚动）

### 1. 顶部 Header

- 应用标题：💰 个人财务分配器
- 简短副标题说明用途

### 2. 月度收入输入区

- 单个数字输入框，前置 ¥ 符号
- 用户输入后**实时触发**整页重新计算（无需点击按钮）
- 最后一次输入的金额持久化到 `localStorage`

### 3. 分配结果卡片区

- 每个一级分类对应一张卡片，显示：
  - 分类名称 + 颜色标签
  - 占比百分比
  - 本月分配金额（粗体，醒目）
  - 二级分类列表：名称 + 对应金额
- 卡片区顶部显示一条**比例进度条**，各一级分类按颜色比例可视化
- 如果所有一级分类比例之和不足 100%，进度条末尾显示灰色"未分配"区域

### 4. 分类设置区（可折叠面板）

- 默认折叠，点击展开/收起
- 功能包括：
  - **新增一级分类**：输入名称、选择颜色、设置比例（%）、设置预期年化收益率（%）
  - **编辑一级分类**：就地修改名称、比例、颜色、收益率
  - **删除一级分类**：带确认提示
  - **新增/编辑/删除二级分类**：在对应一级分类下操作，设置名称和占一级分类的子比例（%）
- 一级分类比例总和**实时校验**：超过 100% 时对应输入框高亮红色并显示警告文字
- 所有改动**即时自动保存**到 `localStorage`，无需手动保存按钮

### 5. 里程碑卡片区

- 标题：📈 长期资产预测
- 展示 **1年 / 3年 / 5年 / 10年 / 20年** 共 5 张卡片
- 每张卡片显示：
  - 年限标签（如"5 年后"）
  - 预计累计资产总额（大字号，醒目）
- 卡片下方注明假设条件：按当前月收入固定不变，各类别独立复利计算

## 数据模型

`localStorage` 中存储的 JSON 结构（key: `finance-config`）：

```json
{
  "lastIncome": 15000,
  "categories": [
    {
      "id": "abc123",
      "name": "存款",
      "percentage": 30,
      "color": "#2563eb",
      "expectedReturn": 2.0,
      "subcategories": [
        { "id": "def456", "name": "活期备用金", "percentage": 10 },
        { "id": "ghi789", "name": "定期存款", "percentage": 20 }
      ]
    }
  ]
}
```

**字段说明：**
- `categories[].percentage`：该分类占**总收入**的比例（0-100）
- `categories[].expectedReturn`：年化预期收益率，单位 %（如 `2.0` 表示 2%/年）
- `subcategories[].percentage`：该二级分类占**所属一级分类金额**的比例（0-100）

**初始状态：** 首次打开时 `localStorage` 为空，设置面板默认展开，引导用户添加第一个分类。

## 计算逻辑

### 月度分配

```
一级分类金额 = 月收入 × (一级比例 / 100)
二级分类金额 = 一级分类金额 × (二级子比例 / 100)
```

### 里程碑复利

假设每年投入固定（月收入 × 12），各分类独立计算后汇总：

```
单分类年投入 = 月收入 × 12 × (该分类比例 / 100)
单分类 n 年资产 = 年投入 × ((1 + r)^n - 1) / r
  其中 r = expectedReturn / 100

n 年总资产 = Σ 各分类 n 年资产
```

特殊情况：当 `r = 0`（年化0%）时，`n 年资产 = 年投入 × n`（避免除以零）。

## 视觉风格

- **配色方案：** 浅色简洁风，白色/浅灰背景（`#f8fafc`），卡片白底带浅色边框和微弱阴影
- **字体：** 系统默认 sans-serif 字体族，或引入 Inter（Google Fonts）
- **颜色编码：** 每个分类有独立颜色，卡片顶部色条 / 进度条使用该颜色区分
- **交互反馈：** 输入框 focus 状态蓝色边框，删除按钮 hover 变红，卡片 hover 轻微上浮阴影
- **响应式：** 分配卡片区在宽屏（≥768px）以 2-3 列网格显示，移动端单列

## 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 所有分类比例之和 < 100% | 进度条末尾显示灰色"未分配"区域，不报错 |
| 所有分类比例之和 > 100% | 高亮超出的输入框，显示红色警告文字，里程碑区显示"-" |
| 月收入为空或0 | 所有金额显示 ¥0，里程碑显示 ¥0 |
| localStorage 为空 | 设置面板默认展开，显示"添加第一个分类"引导提示 |
| 分类无二级分类 | 不显示二级分类列表区域，卡片仅显示一级信息 |
| 年化收益率为 0 | 里程碑按线性计算（年投入 × 年数），避免除以零 |
