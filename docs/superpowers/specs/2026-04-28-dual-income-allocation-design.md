# 双人收入与资金分配结算方案 (Design Spec) - V2

## 1. 概述 (Overview)
在原有的资金池与结算基础上，引入**「绝对金额的固定支出」**与**「百分比的投资分配」**分离的机制。
真实生活中的支出（如房贷、买菜）通常是绝对金额，而投资则是对**扣除支出后的剩余资金**进行百分比分配。本系统将根据成员建立专属的固定支出区域，并自动计算剩余资金池，最后进行统一资金结算。

## 2. 数据结构设计 (Data Model)

### 2.1 成员池与固定支出 (Members & Expenses)
`members` 数组中的每个成员新增一个 `expenses` 数组，用于记录该成员负责的绝对金额支出。
```json
{
  "members": [
    { 
      "id": "m1", 
      "name": "刘", 
      "income": 10000,
      "expenses": [
        { "id": "e1", "name": "房贷", "amount": 5000 },
        { "id": "e2", "name": "宽带", "amount": 100 }
      ]
    },
    { "id": "m3", "name": "小荷包", "income": 0, "expenses": [ { "name": "买菜", "amount": 3000 } ] }
  ],
  "categories": [ ... ] // 投资理财类的百分比分配（作用于剩余金额）
}
```

### 2.2 投资分类指派 (Category Assignment)
维持先前的设计，每个 `Category` 和 `Subcategory` 都有 `assigneeId`，用于指定百分比投资的执行人。

## 3. UI 界面更新 (UI Components)

### 3.1 成员与收入设置 (保持原样)
顶部配置成员名称和他们的实际收入，计算出「合并总收入」。

### 3.2 固定支出编辑区 (新增)
- 在合并总收入下方，新增一个块：**「本月固定支出 (绝对金额)」**。
- 按成员名字分组展示（例如：一个框是“刘的专属支出”，里面列出房贷等）。
- 每个成员分组下有一个「+ 添加支出项目」的按钮。
- 输入绝对金额。
- 底部显示：「扣除支出后的剩余可支配资金 = XXXX」。

### 3.3 百分比分配面板 (逻辑变更)
- 原先的分类占比（以及条形图、分类卡片金额）将**仅作用于「剩余可支配资金」**，而不是总收入。
- 配置项仍然带有「执行人」下拉框。

### 3.4 结算卡片区 (逻辑合并)
- **计算逻辑：**
  1. `成员任务总额 = 该成员名下的所有固定支出总和 + 该成员被指派的投资分类金额总和`
  2. `净差额 = 成员实际收入 - 成员任务总额`
  3. 净差额 > 0 为付款方，< 0 为收款方。
- 结算面板输出明确的转账动作指令。

## 4. 边界情况
- **固定支出大于总收入**：剩余可支配资金将计算为 0，下方的百分比投资卡片全部显示为 0。结算系统依然能正确算出差额（即有人需要借钱或透支）。
- **删除成员**：删除成员时，该成员名下的所有固定支出一并被删除，清理逻辑与之前一致。
